## 2. 보안 그룹 설정 이유

### 첫 번째 ssh : 개인 로컬 ip 설정

이유 : pem키를 가지고 ssh에 로그인 하는 과정은 보안과 직접적으로 관련이 되어 있기 때문에
지정된 ip에서만 접속할 수 있게 개인 로컬 pc에 할당된 ip로 지정

- 깃 허브 액션을 이용하게 되면 해당되는 ip로 변경

### 두 번째 HTTP : 모든 ip

누구나 접속해서 홈페이지를 구경할 수 있게 열어놓음

### 세 번째 3000포트 : 모든 ip

누구나 프론트엔드에 접속해서 사용할 수 있게 열어놓음

### 네 번째 8080포트 : 모든 ip

프론트엔드가 백엔드로부터 데이터를 받아올 수 있게 접속을 허용함
단, spring security로 보안 정책을 걸어놔서 보안에 대비해야 할 필요가 있음
