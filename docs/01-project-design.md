# 프로젝트 설계 문서

## 1. 퀴즈개수를 정한만큼 받아와서 풀어보는 서비스

## 2. 전체 흐름
첫 화면에서 회원가입&로그인 혹은 guest로 진행할지 선택 > 
메인 화면에 퀴즈 종류와 개수를 적어서 퀴즈 풀기 버튼 누름 >
서버측에서 문제를 보내서 화면에 퀴즈를 보내줌 >
로그인한 유저라면 맞춘 개수만큼 score로 기록

## 3. 프론트엔드 역할
 
- 문제 답변을 입력받으면 정답과 비교 후 오답시 정답 표시

## 4. 백엔드 역할

- 유저 객체 저장 및 퀴즈 객체 저장
- 유저 회원가입 로직
- 퀴즈 crud역할
- ROLE을 부여해서 권한 제어
